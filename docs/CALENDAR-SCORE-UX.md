# Calendar Score UX Specification

> 목적: 날짜 선택 단계에서 점수/추천도를 직관적으로 보여주는 캘린더 UX를 고정한다.
> 마지막 업데이트: 2026-04-21

---

## 1. 적용 화면

- 스코어 화면 (`/score`)
- 장소 추천 화면 (`/place`)

---

## 2. UX 목표

- 날짜 탐색 속도 향상 (좋은 날짜를 한눈에 발견)
- 선택 전 비교 가능 (점수/등급 시각화)
- 선택 후 확신 제공 (평균 점수·등급 요약)

---

## 3. 정보 구조

날짜 셀 최소 표시:
- 일자 (`1`, `2`, ...)
- 추천도(등급 또는 점수)
- 등급 텍스트 (`강력추천/추천/보통/비추천`)

스코어 화면 추가 구조:
- 도시 선택 전: 캘린더 비노출(선택 유도 상태)
- 도시 선택 후: 캘린더 노출
- 단위 토글: 월별 / 주차별 / 일별 추천도 뷰

요약 영역:
- 선택 범위 (`10일~14일`)
- 평균 점수
- 평균 등급
- 선택 일수

---

## 4. 등급 규칙

- `80~100`: 강력추천 (`excellent`)
- `60~79`: 추천 (`good`)
- `40~59`: 보통 (`fair`)
- `0~39`: 비추천 (`poor`)

---

## 5. 시각 규칙

- 색상만으로 의미 전달 금지 (라벨 동시 표기)
- 셀 상태
  - 기본
  - 시작일 (`is-start`)
  - 종료일 (`is-end`)
  - 범위 내 (`in-range`)
  - 비활성(`disabled`)
- 호버/포커스 시 시각적 피드백 제공

---

## 6. 상호작용 규칙

- 1차 클릭: 시작일 선택
- 2차 클릭: 종료일 선택
- 3차 클릭: 새 시작일로 초기화
- 시작일 > 종료일 입력 시 자동 정렬(min~max)
- 월 전환 시 선택 범위 초기화(기본 정책)
- 주차 뷰 선택 시: 주차 단위 하이라이트 + 해당 주 추천도 요약
- 일별 뷰 선택 시: 셀 상세(추천 이유 요약 툴팁/패널) 열기

---

## 7. 반응형 규칙

- Desktop/Tablet: 7열
- Mobile: 4열(또는 7열 축소형 중 택1)
- 모바일에서는 요약 영역을 캘린더 하단 고정 배치

---

## 8. 접근성 규칙

- 캘린더: `role="grid"`
- 셀: `role="gridcell"`
- 각 셀 `aria-label` 예시: `"3일, 69점, 추천"`
- 키보드 조작
  - 방향키 이동
  - Enter/Space 선택
- 포커스 링 필수

---

## 9. 데이터 규칙

- 점수 데이터 소스 우선순위
  1. 목적별 점수(`activity_weather_score`) - 장소추천
  2. 일반 기후 점수(`climate_normals` 파생) - 스코어
- 데이터 없음
  - `N/A` 표기
  - 선택 불가 처리

---

## 10. 에러/예외 처리

- 월 데이터 조회 실패: 오류 배너 + 재시도
- 일부 날짜 누락: 셀 비활성 + 툴팁 "데이터 없음"
- 윤년(2/29): 서비스 정책에 따라 2/28 통계 복제 또는 비활성
- 도시 미선택: 캘린더 영역 대신 도시 선택 가이드 표시
- 도시 변경 직후: 이전 선택 범위 초기화 + 스켈레톤 재노출

---

## 13. 상태 전이 명세 (스코어 캘린더)

- `idle` (도시 미선택)
  - 표시: 도시 선택 CTA
  - 이벤트: `city_selected` → `loading`
- `loading`
  - 표시: 캘린더/요약 스켈레톤
  - 이벤트: `fetch_success` → `ready`, `fetch_empty` → `empty`, `fetch_error` → `error`
- `ready`
  - 표시: 월/주/일 추천도 + 범위 선택
  - 이벤트: `mode_changed` → `loading` (재계산), `month_changed` → `loading`
- `empty`
  - 표시: "추천 데이터 없음" + 인접 월/주 CTA
  - 이벤트: `month_changed` 또는 `city_changed` → `loading`
- `error`
  - 표시: 오류 문구 + 재시도
  - 이벤트: `retry` → `loading`

---

## 11. 계측 이벤트(권장)

- `calendar_opened`
- `calendar_day_selected`
- `calendar_range_selected`
- `calendar_month_changed`
- `calendar_apply_clicked`

필드 예시:
- `screen`, `year`, `month`, `start_day`, `end_day`, `avg_score`, `avg_grade`
