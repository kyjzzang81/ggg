# CLIMATE QA Checklist

> 목적: 출시 직전 수동 검증 항목을 통일해 회귀를 빠르게 잡는다.
> 기준: `SCREEN-SPEC.md`, `RESPONSIVE-GUIDE.md`, `CALENDAR-SCORE-UX.md`

---

## 1. 공통 환경

- 브라우저: Chrome 최신, Safari 최신
- 해상도:
  - Mobile: 390x844
  - Tablet: 834x1194
  - Desktop: 1440x900
- 계정:
  - 비로그인
  - 일반 로그인

---

## 2. 핵심 플로우

- [ ] 홈 진입 후 현재 날씨/예보 카드 노출
- [ ] 도시 선택 → 스코어 화면 이동
- [ ] 캘린더 날짜 선택 시 점수/추천도 배지 표시
- [ ] 장소 추천 결과가 목적/날짜에 맞게 변경
- [ ] D-day 생성/수정/삭제 전체 동작
- [ ] 로그아웃 후 보호 라우트 접근 시 로그인 유도

---

## 3. 반응형 검증

- [ ] Mobile: 하단 탭 5개 정상
- [ ] Tablet: 72px 아이콘 사이드바 전환
- [ ] Desktop: 240px 라벨 사이드바 전환
- [ ] 각 뷰포트에서 CTA 버튼 잘림 없음
- [ ] 키보드 오픈 시 입력 UI 깨짐 없음

---

## 4. 날씨 테마 × 레이어 매트릭스

검증 조합:
- 테마: `sunny`, `rain`, `snow`, `heat` (대표 4개 최소)
- 레이어: 기본 / 연인 ON / 가족 ON

체크:
- [ ] 텍스트 대비 WCAG AA 이상
- [ ] 점수 색상과 경고 색상 의미 충돌 없음
- [ ] 가족 ON 시 KidSafetyScore 강조
- [ ] 연인 ON 시 골든아워 정보 강조

---

## 5. 캘린더 UX 검증

- [ ] 날짜 셀에 숫자+등급 동시 표시
- [ ] 범위 선택 시작/끝 시각 강조
- [ ] 비추천 구간 색상 라벨 명확
- [ ] 월 전환 후 선택 상태 유지
- [ ] 잘못된 범위 선택(from > to) 방지

---

## 6. 오류/복구

- [ ] 위치 권한 거부 시 수동 도시 선택 fallback
- [ ] 외부 API 실패 시 캐시 데이터 + 상태 라벨 노출
- [ ] 네트워크 오프라인 시 재시도 버튼 노출
- [ ] 404 경로 접근 시 안전한 fallback

---

## 7. 인증/보안

- [ ] `/dday`, `/mypage` 비로그인 접근 차단
- [ ] 로그인 후 원래 경로(`redirect`) 복귀
- [ ] 계정 전환 시 이전 사용자 데이터 노출 없음
- [ ] 로그아웃 시 민감 캐시 제거

---

## 8. 접근성

- [ ] Tab 키만으로 주요 액션 이동 가능
- [ ] 버튼/아이콘에 aria-label 존재
- [ ] 모달 열림 시 포커스 트랩 동작
- [ ] ESC로 모달 닫힘

---

## 9. 성능 스모크

- [ ] 홈 첫 페인트 3초 내(일반 네트워크)
- [ ] 스크롤 중 프레임 드랍 체감 없음
- [ ] 이미지/차트 로딩 지연 시 skeleton 노출

---

## 10. 결과 기록 템플릿

```md
- 일자:
- 테스터:
- 빌드:
- 실패 항목:
  - [ ] 항목명 / 재현 경로 / 스크린샷 링크
- 차단 이슈:
- 릴리즈 Go/No-Go:
```
