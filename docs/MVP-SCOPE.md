# CLIMATE Phase 1 MVP Scope

> 기준 문서: `PRD.md` (v2.1), `DEV-SPEC.md` (v3.0)  
> 목적: Phase 1에서 **반드시 구현할 범위**와 **명시적으로 제외/유보할 범위**를 확정해 개발 범위를 고정

---

## 1) 스코프 원칙

- Phase 1 목표는 "웹앱 MVP를 빠르게 출시해 핵심 가치 검증"이다.
- 핵심 가치 검증 범위는 아래 3가지다.
  - 날씨/기후 데이터 기반 여행 의사결정 지원
  - 사용자 저장 기능(D-day)으로 재방문 유도
  - 최소한의 수익화/운영 기반(광고 슬롯, 구독 구조를 위한 데이터 기반) 확보
- Phase 1은 **웹 기준**으로 고정하며, 앱(Capacitor) 및 고난도 외부 연동은 원칙적으로 Phase 2+로 이관한다.

---

## 2) Phase 1 포함(필수 구현)

## 2-1. 네비게이션/레이아웃

- 하단 탭 5개: `홈`, `스코어`, `장소 추천`, `주변`, `D-day`
- 모바일 햄버거 사이드 패널: `모드 설정`, `숨은 황금 시즌`, `도시 비교`, `소셜 임팩트`, `마이페이지`
- 반응형 레이아웃(모바일/태블릿/PC)

## 2-2. 핵심 화면 기능

- 홈
  - 현재 날씨, 5일 예보, 핵심 지표 표시
  - 날씨 테마 8종 반영
  - 위치 권한 거부 시 수동 지역 검색 fallback
- Climate Score
  - 목적지 + 시기 입력
  - 점수 카드, 월별 비교 차트, 기본 인사이트
  - 날짜 선택 캘린더에 일자별 Score/등급(추천~비추천) 시각화 (Skyscanner 가격 캘린더 유사 UX)
- 장소 추천
  - 목적 + 시기 기반 상위 추천
  - 기본 정렬(기후 점수 + 목적 달성도)
  - 시기 선택 캘린더에서 날짜별 추천도 배지(강력 추천/추천/보통/비추천) 표시
- 주변(Nearby)
  - 현재 위치 기반 추천 리스트(최소 리스트 뷰)
  - 날씨 조건 기반 카테고리 우선순위
- D-day
  - 일정 생성/수정/삭제
  - D-day 카운트다운 + 날짜 기반 날씨/강수 위험 정보

## 2-3. 모드 레이어(최소 버전)

- 연인/가족 레이어 토글(`modeStore`)
- 독립 화면이 아닌 조건부 렌더링 레이어 방식 유지
- Phase 1에서는 레이어별 "핵심 차별 콘텐츠"만 최소 반영

## 2-4. 데이터/백엔드

- Supabase 기본 스키마 + 핵심 신규 테이블
  - `user_dday_events`
  - `user_sessions`
  - `user_subscriptions` (구조만 우선)
- 예보 갱신 배치 유지(`refresh-forecast`)
- `build-climate-frequency` 또는 동등한 빈도 집계 배치(홈 인사이트 최소 구현용)
- 기본 Auth(로그인 없이 조회, 로그인 시 저장 기능)

## 2-5. 품질 기준(출시 최소선)

- 치명적 에러 없이 주요 플로우 동작:
  - 도시 선택 → 스코어 확인
  - 장소 추천 조회
  - D-day 저장/조회
- 최소 테스트 기준:
  - 핵심 계산 로직 unit test
  - 주요 사용자 플로우 smoke test(수동 또는 자동)

## 3) Phase 1 제외(명시적 비포함)

- Capacitor 기반 iOS/Android 패키징 및 스토어 배포
- Apple Sign In (모바일 심사용)
- 홈 위젯
- 글로벌 확장(다국어 정식 론칭, 대규모 해외 도시 확장)
- B2G 데이터 채널/외부 판매
- B2B SaaS/API 상품화

---

## 4) Phase 1 유보(가능하면 넣되, 일정 압박 시 후순위)

- 결제 실연동(토스/아임포트 빌링키 실제 청구)
- 구독 가격 자동조정 배치 `adjust-subscription-price`의 운영 반영
- KMA TCI 보강 점수
- IQAir 해외 대기질 연동
- Nearby 지도 뷰(리스트 뷰 우선)
- 고급 AI 분석 결론 카드
- 날씨 아카이브 무제한/고급 공유 템플릿
- 소셜 임팩트 대시보드 고도화(초기엔 단순 지표/목업 데이터 가능)

---

## 5) PRD vs DEV-SPEC 정합성 정리

현재 문서 간 해석 충돌 가능성이 있는 부분을 MVP 기준으로 다음과 같이 고정한다.

- 모드 구조: 독립 탭(커플/가족)이 아니라 **레이어 방식** 채택
- 주변(Nearby): Phase 1 포함
- 구독: Phase 1에서 "기능 플래그/데이터 구조" 우선, 실제 청구는 유보 가능
- 위치: 동 단위 GPS는 목표로 하되, 초기에는 안정적인 fallback을 필수 조건으로 둠

## 6) 의사결정 필요 항목 (협업)

아래 항목은 제품/사업 의사결정이 필요하다.

- [ ] 결제 전략: Phase 1에서 실제 결제 오픈 여부
  - A안: 결제 미오픈(구조만 준비)
  - B안: 결제 오픈(토스/아임포트 중 1개 즉시 연동)
- [ ] 추천/점수 계산 고도화 범위
  - A안: 단순 가중치 MVP
  - B안: KMA/IQAir 포함한 확장형
- [ ] Nearby 범위
  - A안: 리스트 뷰만
  - B안: 리스트 + 지도 뷰 동시
- [ ] 소셜 임팩트 화면 데이터 수준
  - A안: 정적/샘플 지표
  - B안: 실시간 집계 연동
- [ ] 구독 가격 조정 배치 운영 반영 시점
  - A안: Phase 1.5
  - B안: Phase 1 포함

## 8) 부록: Phase 1 핵심 성공 조건

- 사용자 1명이 "좋은 여행 시기 찾기 → 비교 → 저장(D-day)"까지 5분 내 완료 가능
- 데이터 신뢰성 체감(현재/예보/기후 점수 간 큰 모순 없음)
- 운영자가 배치 실패/데이터 갱신 상태를 확인할 수 있음

## 9) 추후 개발사항 (PRD에는 있으나 MVP 미포함)

아래 항목은 PRD에는 포함되어 있으나, Phase 1 일정/리스크를 고려해 MVP 범위에서 제외한다.

- 독립 탭형 커플/가족 화면 완성형 운영 (MVP는 레이어 방식 최소 구현)
- 목적별 Climate Score™ 고도화(해수욕/트레킹/사진/미식 가중치 정교 튜닝)
- KMA TCI + IQAir 실서비스 보정 연동 (전 도시 커버리지 기준)
- Nearby 지도 뷰 완성형 + 고급 필터(예산/동선/혼잡도)
- AI 분석 결론 카드 고도화(설명 가능 추천 근거 자동 생성)
- 날씨 아카이브 프리미엄 템플릿/SNS 공유 고급화
- 구독 가격 자동조정 배치의 실제 과금 반영(결제사 정식 연동 포함)
- 소셜 임팩트 실시간 대시보드(외부 파트너/B2G 데이터 연계)
- 홈 위젯 및 네이티브 앱 기능(Capacitor, App Store/Play Store 배포)
- 글로벌 확장(영문 런칭, 해외 도시 대규모 확장, B2B/B2G 채널)
